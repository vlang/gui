interface UI_Tree #

interface UI_Tree {
	id string
	generate(ctx gg.Context) ShapeTree
mut:
	children []UI_Tree
}

UI_Tree is a user defined view. UI_Trees are never displayed directly. Instead a ShapeTree is generated from the UI_Tree. It is the ShapeTree that is used to render the UI.

enum ShapeDirection #

enum ShapeDirection {
	none
	top_to_bottom
	left_to_right
}

ShapeDirection determines if a Shape arranges its child shapes horizontally, vertically or not at all.

enum ShapeType #

enum ShapeType {
	none
	container
	rectangle
	text
	line
	image
}

ShapeType describes the only shapes a Shape can be. All screen rendering is comprised of these ShapeTypes.

enum SizingType #

enum SizingType {
	fit   // Shape is fitted to child shapes
	flex  // Shape can grow or shrink as needed
	fixed // Shape is only the given size.
}

SizingType describes the three sizing modes of GUI

struct Shape #

struct Shape {
pub:
	id        string // asigned by user
	uid       string
	type      ShapeType
	direction ShapeDirection
mut:
	x          f32
	y          f32
	width      f32
	height     f32
	spacing    f32
	sizing     Sizing
	padding    Padding
	fill       bool
	radius     int
	color      gg.Color
	text       string
	lines      []string
	text_cfg   gx.TextCfg
	wrap       bool
	min_width  f32
	min_height f32
	bounds     gg.Rect
}

Shape is the only data structure in GUI used to draw to the screen.

fn (Shape) draw #

fn (shape Shape) draw(ctx gg.Context)

draw Drawing a shape it just that. No decisions about UI state are considered. If the UI state of your view changes, Generate an new view and update the window with the new view. New shapes are generated based on the view (UI_Tree). Data flows one way from view -> shapes or in terms of data structures from UI_Tree -> ShapeTree
